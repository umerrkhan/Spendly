❯ .\venv1\Scripts\Activate.ps1                                                  

● The ! prefix in Claude Code runs commands through bash (Git Bash on Windows),
  not PowerShell — that's why Activate.ps1 keeps failing. Bash also strips
  backslashes, which is why .\venv-1\... becomes .venv-1....

  For a venv in bash, use the bash activation script instead:

  source ./venv-1/Scripts/activate

  (note: lowercase activate, no .ps1 extension — this is the bash variant that
  python -m venv creates alongside the PowerShell one).

  Alternatively, if you want PowerShell activation, run it in a real PowerShell
  terminal outside the ! prompt — the activation only persists in the shell
  where you run it anyway, so activating inside ! wouldn't affect your other
  terminal.
